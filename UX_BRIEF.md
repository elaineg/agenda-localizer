# UX Brief — Agenda Localizer

## 1. Problem statement
Paste your event schedule once and share one link that shows every session in each person's own local time — so nobody does timezone math or shows up at the wrong hour.

## 2. Primary user action
Paste a multi-session agenda and copy a share link. Landing view is the CREATOR editor, pre-loaded with the sample agenda already parsed and localized — the user sees a working before/after the instant the page loads, before typing or clicking anything.

## 3. Emotional tone
Fast and trustworthy — like a clean dev tool. Neutral sans-serif (system/Inter), cool/neutral color temperature (slate + one calm accent for the localized time), comfortable-but-dense spacing so a full agenda reads as a tidy list, not a wall.

## 4. Design decisions (builder must not skip)
- **Live, no-submit parse + structured preview.** As the user edits text or changes source tz, sessions re-render live as cards. Each card shows the **viewer's local time large/primary** and the **source time small/secondary beside it** (e.g. "12:00 PM (your time) · 16:00 UTC"). Always display the detected viewer timezone explicitly near the top (e.g. "Times shown in your timezone: America/New_York"). This is how a timezone tool earns trust.
- **Lenient parser, three row types — never a false error (G1).** The parser must silently accept bare-hour and informal times exactly like `HH:MM`: "8pm", "9am", "7 PM", "4 PM ET", "11 p.m.", "noon", "midnight", with or without a tz token. Each input line resolves to ONE of three rows, all rendered in original order:
  1. **Session card** — has a parseable time (localized as above).
  2. **Note / header row** — has NO time (e.g. "Day 1 — Summit", "lunch", "TBD", a date heading). Render as a visually distinct lightweight row (e.g. smaller, muted, no time column, maybe a subtle divider) — a section label, **NOT** an error. The default is "timeless line = note," not "timeless line = broken."
  3. **Needs-a-time warning** — ONLY when a line clearly looks like it intended a time but couldn't be read. Inline, actionable: "Couldn't read a time here — try `16:00 UTC` or `4 PM ET`." Show a small count ("8 sessions · 1 line needs a time"). **Creator view only — see §5.**
- **Visible timezone correctness (G3) — design for trust, never silently-wrong.** (a) Date-cross badge: when a session's localized time lands on a different calendar day than the source, show a small badge on that card — "+1 day" or "−1 day" — next to the local time. (b) Per-line tz tokens (SGT, JST, CET, PST/PDT, ET, etc.) parse and localize correctly; expand the abbreviation table. (c) If a per-line tz token is unrecognized or conflicts with the source-tz dropdown, FLAG it on the card (a small visible warning chip: "Unknown timezone 'XYZ' — using source tz") — never silently fall back. Confidently-wrong output is the one unforgivable failure for this app.
- **Calendar export on BOTH previews (G4).** The per-session "Add to Google Calendar" + "Download .ics" affordances appear on the creator-side localized preview too, not only the shared view. Same affordance, same place on each card.
- **One-click "Download all sessions (.ics)" — the bulk wedge (G8).** A 6–10-session launch week today means 10 separate clicks; add ONE prominent action that downloads a single multi-VEVENT .ics containing every valid session (reuse the per-session UTC start/end so times are correct for any viewer). This is the PRIMARY bulk action; per-session links stay SECONDARY (unchanged, on each card). **Placement — both surfaces, above the fold:** (a) **Shared/attendee view** — a full-width primary button (sky fill, calendar icon, "📅 Download all sessions (.ics)") sits in the agenda HEADER, directly under the viewer-timezone banner and ABOVE the first session card, so a cold attendee sees "add the whole agenda" without scrolling (see §5 / the panel "added-feature-buried" lesson). (b) **Creator preview** — same button at the TOP of the localized-preview column (desktop right column header; mobile in the compact preview block above the fold), in/beside the same control cluster as "Copy share link" but visually grouped with the preview as a calendar action, not a copy action. **Honesty:** the combined action is .ics DOWNLOAD ONLY — no bulk Google-Calendar button (their template URL is single-event-only); the button label/help text must NOT promise Google Calendar for the bulk case ("Imports into Google Calendar, Apple Calendar & Outlook" is the correct framing — .ics import works everywhere). **Disabled/empty state:** when 0 valid sessions, the button is hidden (preferred) or disabled at 40% opacity with no download — never emits an empty file. **Filename:** derive from the agenda title via the existing slug logic, e.g. `launch-week-2026.ics`, fallback `agenda.ics`. On click, give a brief inline confirmation under the button ("Downloaded — N sessions added") consistent with the copy-confirmation pattern, so the click registers.
- **Title cleanup (G7).** Strip trailing separators and tz tokens from titles ("Standup @" → "Standup"; "Kickoff SGT" → "Kickoff"). Do NOT strip internal colons — keep "Workshop: Building with AI" readable. Only the time/tz portion is removed; the human-readable label stays intact.
- **Copy confirmation that a hurried user CANNOT miss (H3 — round-2: 2 real testers never saw it).** The current flip works mechanically but is too subtle/brief. On copy, the "Copy share link" button must change ALL THREE channels at once so the change registers peripherally: (a) **color** — flip to the success/green fill (not just text); (b) **icon** — show a check; (c) **label** — "✓ Copied!". Hold the changed state for **~2s** (not a 50ms blink), then revert. Plus a **brief inline confirmation directly beside/below the button** — "Link copied — paste it anywhere" — that appears in the same 2s window. The button label change is the source of truth (survives re-render via component state + aria-live, NOT a toast that async-clipboard/re-parse clobbers); the inline line is reinforcement. Confirmation must fire on BOTH the native clipboard path and the fallback path. Make the visual delta large enough to see without looking straight at the button.
- **Copy as table — analyst-friendly export (H4 — round-2: Wen, data analyst).** Beside the "Copy share link" button (same control cluster, secondary-weight so it doesn't compete), add a **"Copy as table"** button that copies the localized agenda to the clipboard as tab-separated rows (TSV — pastes clean into Sheets/Excel and Slack), with a header row. Columns, in order: **Session · Local time · Local date · Source time · Source timezone**. Use the SAME unmistakable copy confirmation as above (color + check + "✓ Copied table!" + inline "Pasted into a sheet? Columns are tab-separated."). It must read as a deliberate feature for people who audit schedules in a spreadsheet, not a bolt-on. Client-side only — never touches a server (on-brand with the privacy prop). Lives on the creator view; optional on the shared view.
- **Privacy line, plain language (G5).** Near the editor/share button, one quiet line: "Runs entirely in your browser; nothing is uploaded — your agenda travels inside the link itself." Add a smaller adjacent note: "The share link contains your full agenda." Calm/reassuring, not a scare banner.

## 5. Shared / ATTENDEE view (the differentiator + growth surface)
Opening a share URL shows a polished READ-ONLY page (no editor, no paste box): a clean agenda title area, the viewer-timezone banner, the **primary "📅 Download all sessions (.ics)" button (G8) — full-width, directly below the banner and above the first card, so it is the first interactive element a cold attendee sees without scrolling**, then the session cards (local time primary, source secondary) each with "Add to Google Calendar" + "Download .ics". The bulk button reads as the headline action; per-session links are the secondary fallback for "just this one." Must look finished and trustworthy to a stranger who's never seen the app. A subtle, non-defacing footer: "Made with → Agenda Localizer" linking to the empty editor. Distinguish modes purely by presence of agenda data in the URL hash: hash present = shared read-only; no hash = creator editor.

**Human-readable share link (H5 — round-2: Rob, Elena — a titleless ~233-char blob).** The share URL must carry a legible slug derived from the agenda's title (its first heading / first session title), so a link pasted into Slack or email shows something meaningful — e.g. `/agenda/launch-week-2026#eyJ...payload`. Build the slug from the title: lowercase, dashes, strip tz/time tokens, cap ~6 words; fall back to `agenda` when there's no title. **Hard constraint — the round-trip MUST still work from the encoded payload ALONE.** The slug is cosmetic and is regenerated/ignored on load; never parse data out of it, never require it to match. An edited or hand-truncated slug on the same payload must still decode to the same agenda. The slug updates live as the title changes, but the existing copy-confirmation behavior and decode path are unchanged.

**NEVER leak parse warnings here (G2 — existential for the growth surface).** The shared view shows zero "couldn't read a time" warnings, ever. Per row type: session cards render normally (incl. date-cross badges, which DO belong here — they're correctness, not errors); note/header rows render as the same muted note rows as in the editor; lines that were flagged as needs-a-time in the creator view are simply omitted from the shared view. The attendee sees a clean, finished agenda — no yellow, no "broken" feel.

## 6. 5-second check (above the fold, cold visitor)
- **Headline:** "Your agenda, in everyone's timezone."
- **Subtitle:** "Paste your sessions, share one link — each person sees their own local time."
- **Primary element:** the sample agenda already parsed into localized session cards (local time primary, source secondary), with the editor and "Copy share link" button visible. A "Load sample agenda" button is present but the sample is pre-loaded so value is obvious with zero clicks.
- **Bulk action visible cold:** the "📅 Download all sessions (.ics)" button is rendered above the fold on both the creator preview and — critically — the shared attendee view, where it appears immediately under the timezone banner before any card. A cold visitor sees "get the whole agenda into my calendar" without scrolling or being prompted (panel lesson "added-feature-buried": the new bulk action must be discoverable on first paint).

## 7. Empty / error / edge / mobile
- True empty editor (creator clears text): show a one-line worked example + a "Load sample agenda" button, never a blank box.
- Unparseable share URL / corrupt hash: friendly message + "Make your own" CTA, no crash.
- Mobile (attendees open links on phones): single-column cards, local time stays the largest element, calendar buttons full-width and tappable; the viewer-timezone banner stays visible.
- **Mobile CREATOR editor — proof above the fold (G6).** On a phone, the sender must see the localized preview working BEFORE copying, without scrolling past the textarea. Show a compact/collapsed localized summary above the fold (e.g. the first 1–2 localized cards + the "N sessions" count, expandable), so the sender sees proof the link works before hitting Copy. The editor textarea can be shorter on mobile to make room.
